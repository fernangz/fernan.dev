Web Security Essentials

Protect your users and applications from common threats

Introduction

Web security is critical for protecting user data and maintaining trust. FrontEnd developers play a crucial role in preventing common attacks like XSS, CSRF, and clickjacking. This guide covers essential security practices every developer should implement.

Cross-Site Scripting (XSS)

XSS attacks inject malicious scripts into trusted websites:

Types of XSS

  • Reflected: Malicious script reflected off web server
  • Stored: Malicious script stored on target server
  • DOM-based: Vulnerability in client-side code

Prevention

// NEVER use innerHTML with user input
element.innerHTML = userInput; // DANGEROUS

// Use textContent instead
element.textContent = userInput; // SAFE

// Or escape HTML entities
function escapeHTML(str) {
    return str.replace(/[&<>'"]/g, 
        tag => ({
            '&': '&',
            '<': '<',
            '>': '>',
            "'": ''',
            '"': '"'
        }[tag]));
}

// Frameworks handle this automatically
// React, Vue, Angular escape by default

Cross-Site Request Forgery (CSRF)

CSRF tricks users into executing unwanted actions:

Prevention

  • Use CSRF tokens for state-changing requests
  • Implement SameSite cookie attribute
  • Verify Origin and Referer headers
// Set SameSite cookie
Set-Cookie: session=abc123; SameSite=Strict; Secure

// Or Lax for less strict requirement
Set-Cookie: session=abc123; SameSite=Lax; Secure

// CSRF token in forms
<input type="hidden" name="csrf_token" value="abc123">

Content Security Policy (CSP)

CSP prevents XSS by controlling resource loading:

// HTTP Header
Content-Security-Policy: default-src 'self'; 
    script-src 'self' https://trusted.com;
    style-src 'self' 'unsafe-inline';
    img-src 'self' data: https:;
    connect-src 'self' https://api.example.com;
    frame-ancestors 'none';

// Meta tag alternative
<meta http-equiv="Content-Security-Policy" 
    content="default-src 'self'">

Common Directives

  • default-src - Fallback for other directives
  • script-src - Allowed JavaScript sources
  • style-src - Allowed CSS sources
  • img-src - Allowed image sources
  • connect-src - Allowed fetch/XHR endpoints
  • frame-ancestors - Clickjacking protection

HTTPS and Secure Headers

Always use HTTPS and security headers:

// Essential security headers
Strict-Transport-Security: max-age=31536000; includeSubDomains
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
X-XSS-Protection: 1; mode=block
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: geolocation=(), microphone=()

Input Validation

Validate and sanitize all user input:

// Validate email
function isValidEmail(email) {
    return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
}

// Validate and sanitize
function sanitizeInput(input) {
    return input
        .trim()
        .replace(/[&<>"']/g, '');
}

// Use validation libraries
import { z } from 'zod';

const userSchema = z.object({
    email: z.string().email(),
    age: z.number().min(18),
    name: z.string().min(1).max(100)
});

Authentication Security

  • Use secure password hashing (bcrypt, Argon2)
  • Implement rate limiting on login attempts
  • Store tokens securely (httpOnly cookies)
  • Implement proper session timeout
  • Use multi-factor authentication
// Secure cookie for tokens
res.cookie('token', token, {
    httpOnly: true,  // Not accessible via JavaScript
    secure: true,    // HTTPS only
    sameSite: 'strict',
    maxAge: 3600000  // 1 hour
});

Dependency Security

Keep dependencies secure and updated:

  • Regularly update packages
  • Use npm audit or yarn audit
  • Check for known vulnerabilities
  • Pin dependency versions
# Check for vulnerabilities
npm audit
npm audit fix

# Use Snyk for continuous monitoring
npx snyk test

Security Checklist

  • ✅ Use HTTPS everywhere
  • ✅ Implement CSP headers
  • ✅ Escape all user input
  • ✅ Use CSRF tokens
  • ✅ Set secure cookie flags
  • ✅ Validate input client and server-side
  • ✅ Keep dependencies updated
  • ✅ Use security headers
  • ✅ Implement rate limiting
  • ✅ Use httpOnly cookies for tokens

Security Testing Tools

  • OWASP ZAP: Free security scanner
  • Security Headers: Check security headers
  • Mozilla Observatory: Comprehensive security testing
  • npm audit: Check dependency vulnerabilities

Conclusion

Security is not optional—it's essential. Implement these practices from the start, stay informed about new threats, and regularly audit your applications. A secure application protects your users and your reputation.
