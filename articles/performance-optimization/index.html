Performance Optimization

Make your websites faster and improve Core Web Vitals

Introduction

Website performance directly impacts user experience, conversions, and search rankings. A 1-second delay can reduce conversions by 7%. Google's Core Web Vitals are now ranking factors. This guide covers practical techniques to make your websites faster.

Core Web Vitals

Google measures three key metrics:

  • LCP (Largest Contentful Paint): Loading performance. Target: <2.5s
  • FID (First Input Delay): Interactivity. Target: <100ms
  • CLS (Cumulative Layout Shift): Visual stability. Target: <0.1

Optimize Images

Images are often the largest assets on a page:

  • Use modern formats (WebP, AVIF) with fallbacks
  • Implement responsive images with srcset
  • Lazy load off-screen images
  • Compress images appropriately
<img 
    src="image.webp"
    srcset="image-400.webp 400w, image-800.webp 800w"
    sizes="(max-width: 600px) 400px, 800px"
    alt="Description"
    loading="lazy"
    decoding="async"
    width="800"
    height="600"
>

Minimize JavaScript

JavaScript blocks rendering and delays interactivity:

  • Remove unused JavaScript (tree-shaking)
  • Code-split large bundles
  • Use async or defer for scripts
  • Consider vanilla JS over heavy frameworks
<!-- Defer non-critical scripts -->
<script src="app.js" defer></script>

<!-- Async for independent scripts -->
<script src="analytics.js" async></script>

Optimize CSS Delivery

CSS blocks rendering too:

  • Critical CSS inline in <head>
  • Defer non-critical CSS
  • Remove unused CSS
  • Minify CSS files
<!-- Critical CSS inline -->
<style>
    /* Above-the-fold styles only */
    header, nav, .hero { ... }
</style>

<!-- Non-critical CSS deferred -->
<link rel="preload" href="styles.css" as="style" onload="this.rel='stylesheet'">

Use a CDN

Content Delivery Networks serve assets from servers closer to users:

  • Reduced latency for global audiences
  • Better caching and compression
  • HTTP/2 support
  • DDoS protection

Enable Compression

Compress text-based assets:

  • Gzip: Widely supported, good compression
  • Brotli: Better compression, modern browsers
# Nginx configuration
gzip on;
gzip_types text/plain text/css application/json application/javascript;
gzip_min_length 1000;

Leverage Browser Caching

Cache static assets to reduce repeat visits:

# Cache static assets for 1 year
Cache-Control: public, max-age=31536000

# HTML should not be cached long
Cache-Control: no-cache

Use content hashes in filenames for cache busting.

Reduce Layout Shifts

Prevent CLS by reserving space:

  • Specify width and height on images
  • Reserve space for ads and embeds
  • Use aspect-ratio CSS property
  • Avoid inserting content above existing content
/* Reserve space for images */
img {
    width: 100%;
    height: auto;
    aspect-ratio: 16 / 9;
}

/* Reserve space for embeds */
.embed-container {
    aspect-ratio: 16 / 9;
}

Optimize Web Fonts

Fonts can significantly impact performance:

  • Use font-display: swap
  • Subset fonts to include only needed characters
  • Preload critical fonts
  • Consider system fonts for body text
@font-face {
    font-family: 'MyFont';
    src: url('font.woff2') format('woff2');
    font-display: swap;
    unicode-range: U+0000-00FF; /* Latin only */
}

<link rel="preload" href="font.woff2" as="font" crossorigin>

Performance Testing Tools

  • Lighthouse: Built into Chrome DevTools
  • PageSpeed Insights: Google's online tool
  • WebPageTest: Advanced testing from multiple locations
  • Chrome DevTools Performance tab: Real-time profiling

Conclusion

Performance optimization is an ongoing process. Start with measuring (Lighthouse), identify bottlenecks, implement fixes, and measure again. Small improvements compound into significant gains.
