CSS Custom Properties

Use CSS variables for maintainable, dynamic stylesheets

Introduction

CSS Custom Properties (CSS variables) allow you to store values that can be reused throughout your stylesheet. They're native to CSS, support theming, and can be manipulated with JavaScript—making them far more powerful than preprocessor variables.

Basic Syntax

Custom properties start with -- and are accessed with var():

:root {
    --primary-color: #3498db;
    --spacing-unit: 8px;
    --font-size-base: 1rem;
}

.button {
    background-color: var(--primary-color);
    padding: var(--spacing-unit);
    font-size: var(--font-size-base);
}

Scoping Variables

Variables cascade like regular CSS:

/* Global scope */
:root {
    --color: blue;
}

/* Component scope */
.card {
    --color: red; /* Overrides global */
}

/* Child inherits */
.card .text {
    color: var(--color); /* red */
}

/* Outside component */
.text {
    color: var(--color); /* blue */
}

Fallback Values

Provide fallbacks for unsupported browsers or missing variables:

.element {
    color: var(--text-color, #333);
    /* Multiple fallbacks */
    background: var(--bg, var(--fallback-bg, white));
}

Theming with Custom Properties

Create light and dark themes easily:

/* Light theme (default) */
:root {
    --bg-color: #ffffff;
    --text-color: #333333;
    --link-color: #0066cc;
}

/* Dark theme */
[data-theme="dark"] {
    --bg-color: #1a1a1a;
    --text-color: #ffffff;
    --link-color: #66b3ff;
}

/* Usage */
body {
    background: var(--bg-color);
    color: var(--text-color);
}

a {
    color: var(--link-color);
}

Toggle theme with JavaScript:

document.documentElement.setAttribute(
    'data-theme', 
    'dark'
);

Responsive Values

Change variables at breakpoints:

:root {
    --font-size: 1rem;
    --spacing: 1rem;
}

@media (min-width: 768px) {
    :root {
        --font-size: 1.125rem;
        --spacing: 1.5rem;
    }
}

@media (min-width: 1024px) {
    :root {
        --font-size: 1.25rem;
        --spacing: 2rem;
    }
}

JavaScript Integration

Read and write custom properties with JavaScript:

// Read a variable
const color = getComputedStyle(document.documentElement)
    .getPropertyValue('--primary-color');

// Write a variable
document.documentElement.style
    .setProperty('--primary-color', '#ff0000');

// Remove a variable
document.documentElement.style
    .removeProperty('--primary-color');

Practical Examples

Spacing system:

:root {
    --space-1: 0.25rem;
    --space-2: 0.5rem;
    --space-3: 0.75rem;
    --space-4: 1rem;
    --space-6: 1.5rem;
    --space-8: 2rem;
}

.card {
    padding: var(--space-4);
    margin-bottom: var(--space-4);
}

.card-header {
    margin-bottom: var(--space-3);
}

Color palette:

:root {
    --color-primary: #3498db;
    --color-secondary: #2ecc71;
    --color-danger: #e74c3c;
    --color-warning: #f1c40f;
    
    --color-text: #333;
    --color-text-light: #666;
    --color-border: #ddd;
}

Browser Support

  • Chrome 49+
  • Firefox 31+
  • Safari 9.1+
  • Edge 15+
  • Not supported in IE11

Use PostCSS custom-properties plugin for legacy support.

Best Practices

  • Define variables in :root for global scope
  • Use descriptive, consistent naming
  • Group related variables together
  • Document your design tokens
  • Use fallbacks for critical properties

Conclusion

CSS Custom Properties are essential for modern CSS. They enable theming, reduce repetition, and create more maintainable stylesheets. Start using them today for a more efficient workflow.
