ECMAScript, JavaScript or JS

Understanding the relationship between ECMAScript and JavaScript

Introduction

JavaScript, originally named Mocha, was created in just 10 days by Brendan Eich in May 1995 while he was working at Netscape Communications. It was first released in Netscape Navigator 2.0 as LiveScript, but was quickly renamed to JavaScript (capitalizing on Java's popularity at the time). Despite its name, JavaScript is not directly related to Java—the name was primarily a marketing decision.

The terms JavaScript, ECMAScript, and JS are often used interchangeably, but understanding their differences is important for any FrontEnd developer.

What is ECMAScript?

ECMAScript is the standardized specification that defines how JavaScript should work. It's maintained by ECMA International (European Computer Manufacturers Association) through the TC39 committee, which consists of representatives from various browser vendors, companies, and the open-source community.

Think of ECMAScript as the blueprint or recipe, while JavaScript is the implementation of that specification in web browsers and other environments. Other implementations include:

  • JavaScript: Mozilla's implementation (Firefox) and the most widely used
  • V8: Google's implementation (Chrome, Node.js, Deno)
  • JavaScriptCore: Apple's implementation (Safari)
  • Chakra: Microsoft's legacy implementation (Edge Legacy)

JavaScript vs ECMAScript

The relationship between JavaScript and ECMAScript can be confusing:

  • ECMAScript: The specification/standard that defines the language syntax, types, statements, objects, and more
  • JavaScript: The most popular implementation of ECMAScript (used in browsers and Node.js)
  • JS: Common abbreviation for JavaScript

An analogy: ECMAScript is like the HTTP specification, while JavaScript is like Apache or Nginx (implementations of that specification).

ECMAScript Versions

ECMAScript has evolved through several versions:

  • ES1 (1997): First standardized version
  • ES2 (1998): Editorial changes to keep spec aligned with ISO standards
  • ES3 (1999): Added regular expressions, better error handling, new array methods
  • ES4 (2008): Abandoned due to complexity and disagreements
  • ES5 (2009): Added strict mode, JSON support, array methods (forEach, map, filter, etc.)
  • ES6/ES2015: Major update with classes, modules, arrow functions, let/const, promises, and more
  • ES2016-ES2024: Annual releases with incremental improvements

After ES6, TC39 moved to annual releases to deliver features faster and more incrementally.

Key Features by Version

ES5 Features (2009)

  • 'use strict' - Strict mode for catching errors
  • Array.forEach(), Array.map(), Array.filter(), Array.reduce()
  • JSON.parse() and JSON.stringify()
  • Object.create(), Object.defineProperty(), Object.keys()
  • Date.now(), Function.bind()

ES6/ES2015 Features

  • let and const - Block-scoped variables
  • Arrow functions - () => {}
  • Classes - class MyClass {}
  • Template literals - `Hello ${name}`
  • Destructuring - const {a, b} = obj
  • Modules - import and export
  • Promises - Native async handling
  • Default parameters - function f(x = 1) {}
  • Rest/Spread operators - ...
  • Map, Set, WeakMap, WeakSet
  • Generators - function*
  • Symbols - Unique primitive values

ES2016 Features

  • Array.prototype.includes()
  • Exponentiation operator - **

ES2017 Features

  • Async/await - Cleaner async code
  • Object.values(), Object.entries()
  • String padding - padStart(), padEnd()
  • Trailing commas in function parameters

ES2018 Features

  • Async iteration - for await...of
  • Object rest/spread - {...obj}
  • Promise.prototype.finally()
  • RegExp improvements (lookbehind, named groups, dotAll)

ES2019 Features

  • Array.flat(), Array.flatMap()
  • Object.fromEntries()
  • String trimStart(), trimEnd()
  • Optional catch binding
  • Symbol.description

ES2020 Features

  • Optional chaining - obj?.prop
  • Nullish coalescing - ??
  • BigInt - Arbitrary precision integers
  • Promise.allSettled()
  • globalThis - Universal global object
  • Dynamic import - import()

ES2021 Features

  • Logical assignment operators - &&=, ||=, ??=
  • Numeric separators - 1_000_000
  • String.replaceAll()
  • Promise.any()
  • WeakRef and FinalizationRegistry

ES2022 Features

  • Class fields and private methods - #private
  • Top-level await
  • Array.at()
  • RegExp match indices - d flag
  • Error.cause - Better error chaining

ES2023 Features

  • Array.findLast(), Array.findLastIndex()
  • Array.toReversed(), Array.toSorted(), Array.toSpliced()
  • Array.with() - Immutable array updates
  • Hashbang grammar - #!/usr/bin/env node

JavaScript Basics

JavaScript's syntax is C-style, with curly-brace blocks and semicolons to end statements (though ASI - Automatic Semicolon Insertion - often makes them optional). Variables can be declared using var, let, or const, each with different scoping rules:

  • var: Function-scoped, can be redeclared, hoisted with undefined initial value
  • let: Block-scoped, cannot be redeclared in same scope, hoisted but in temporal dead zone
  • const: Block-scoped, cannot be reassigned, must be initialized at declaration

JavaScript Data Types

JavaScript has 8 data types:

  • Primitive types (7):
    • Number - Integers and floating-point numbers
    • String - Text data
    • Boolean - true or false
    • Null - Intentional absence of value
    • Undefined - Variable declared but not assigned
    • Symbol - Unique identifier (ES6)
    • BigInt - Arbitrary precision integers (ES2020)
  • Reference type (1):
    • Object - Including Arrays, Functions, Dates, RegExp, etc.

The TC39 Proposal Process

New features go through a 5-stage process before becoming part of ECMAScript:

  • Stage 0 (Strawman): Initial idea or proposal
  • Stage 1 (Proposal): Worth pursuing, problem identified
  • Stage 2 (Draft): Initial spec text, API defined
  • Stage 3 (Candidate): Complete spec, needs implementation feedback
  • Stage 4 (Finished): Ready to be included in the standard

You can track proposals at github.com/tc39/proposals.

Best Practices

  • Use const by default, let when reassignment is needed
  • Avoid var in modern codebases
  • Use transpilers (Babel, TypeScript) for broader browser support
  • Use polyfills for missing features in target environments
  • Stay updated with new ECMAScript proposals and releases
  • Check browser support on caniuse.com before using new features
  • Understand the TC39 proposal process for new features

Conclusion

JavaScript continues to evolve, with new features and improvements being added regularly through the ECMAScript standard. Understanding the relationship between ECMAScript and JavaScript helps you write better, more informed code. Whether you're using the latest ES2023 features or maintaining legacy ES5 code, knowing the specification helps you navigate the JavaScript ecosystem more effectively.

As always, the future of JavaScript looks bright and exciting, with ongoing improvements to the language and its ecosystem. Stay curious, keep learning, and embrace the evolution of the web's most popular programming language.
